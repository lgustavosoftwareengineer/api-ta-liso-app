Diagrama de Entidades — Entity Relationship Diagram (ERD)
👤 users Entidade Central
🔑 id UUID PK
email VARCHAR(255) UNIQUE
name VARCHAR(100) NOT NULL
created_at TIMESTAMPTZ DEFAULT NOW()
🔐 login_tokens Auth
🔑 id UUID PK
🔗 user_id UUID FK → users
token VARCHAR(6) NOT NULL
expires_at TIMESTAMPTZ TTL 10min
used BOOLEAN DEFAULT false
created_at TIMESTAMPTZ DEFAULT NOW()
⚙️ user_settings Config
🔑 id UUID PK
🔗 user_id UUID FK → users
alert_low_balance BOOLEAN DEFAULT true
monthly_reset BOOLEAN DEFAULT true
block_negative_balance BOOLEAN DEFAULT false
updated_at TIMESTAMPTZ DEFAULT NOW()
📂 categories Core
🔑 id UUID PK
🔗 user_id UUID FK → users
name VARCHAR(100) NOT NULL
icon VARCHAR(10) emoji
initial_amount NUMERIC(10,2) orçamento
current_balance NUMERIC(10,2) saldo atual
reset_month SMALLINT lazy reset
reset_year SMALLINT lazy reset
created_at TIMESTAMPTZ DEFAULT NOW()
updated_at TIMESTAMPTZ DEFAULT NOW()
💸 transactions Core
🔑 id UUID PK
🔗 user_id UUID FK → users
🔗 category_id UUID FK → categories
description VARCHAR(255) NOT NULL
amount NUMERIC(10,2) NOT NULL
created_at TIMESTAMPTZ DEFAULT NOW()
📸 category_monthly_snapshots Histórico
🔑 id UUID PK
🔗 category_id UUID FK → categories
month SMALLINT 1–12
year SMALLINT ex: 2025
initial_amount NUMERIC(10,2) orçamento do mês
total_spent NUMERIC(10,2) total gasto
final_balance NUMERIC(10,2) saldo final
created_at TIMESTAMPTZ DEFAULT NOW()
1 : N
users → login_tokens
Um usuário pode ter vários tokens de login gerados. Cada token pertence a um único usuário. ON DELETE CASCADE.
1 : 1
users → user_settings
Cada usuário possui exatamente um registro de configurações, criado automaticamente no cadastro. ON DELETE CASCADE.
1 : N
users → categories
Um usuário pode ter várias categorias de orçamento. Cada categoria pertence a um único usuário. ON DELETE CASCADE.
1 : N
users → transactions
Um usuário pode ter vários lançamentos. Mantido direto no user para consultas rápidas sem JOIN em categorias. ON DELETE CASCADE.
1 : N
categories → transactions
Uma categoria pode ter vários lançamentos. Se a categoria for excluída, category_id fica NULL (SET NULL) preservando o histórico.
1 : N
categories → category_monthly_snapshots
Uma categoria gera um snapshot por mês ao ser resetada. Preserva o histórico de orçamento e gastos mês a mês. ON DELETE CASCADE.
PK — Primary Key (UUID)
FK — Foreign Key
UNIQUE
NOT NULL / DEFAULT
users — entidade central
categories + transactions — core
snapshots — histórico
